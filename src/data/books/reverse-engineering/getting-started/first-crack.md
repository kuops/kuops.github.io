# 破解第一个 CrackMe

## 动手目标

今天结束你会：

1. 用 VS 写一个最简单的密码验证程序
2. 用 x32dbg 打开编译好的 exe
3. 搜索字符串，改一个字节，破解自己的程序

不需要任何基础，照着做就行。

## 准备工具

你需要两样东西：

1. **Visual Studio** — 微软的 IDE，免费社区版就够。安装时勾选"使用 C++ 的桌面开发"
2. **x64dbg** — 免费开源的调试器，去 [x64dbg.com](https://x64dbg.com/) 下载最新版，解压就能用

## 自己写一个 CrackMe

我们先自己写一个最简单的密码验证程序，编译成 exe，然后用 x32dbg 去破解它。

打开 VS，创建一个新的 **空项目**（C++ Empty Project）或 **控制台应用**项目。把主文件内容替换成：

<!-- 📸 截图：VS 创建 C++ 控制台项目 -->

```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <string.h>

int main() {
    char input[64] = {0};
    printf("Enter password: ");
    if (scanf("%63s", input) != 1) {
        return 1;
    }

    if (strcmp(input, "reverse2026") == 0) {
        printf("Correct!\n");
    } else {
        printf("Wrong!\n");
    }

    printf("Press Enter to exit...");
    getchar();
    getchar();

    return 0;
}
```

代码很简单：让你输入密码，和 `"reverse2026"` 比较，对了输出 Correct，错了输出 Wrong。最后加了等待输入，防止控制台窗口一闪而过。

**编译注意事项：**

1. 顶部工具栏把配置改成 **Debug**，平台改成 **x86**（即 Win32，不是 x64）
   - `<!-- 📸 截图：VS 工具栏 Debug | x86 下拉框 -->`
2. 按 **Ctrl+B** 生成解决方案
3. 编译好后，在项目目录的 `Debug/` 文件夹里找到 `.exe` 文件

<!-- 📸 截图：VS 编译成功输出 -->

> **为什么用 x86（32位）？** 32 位程序的汇编更简单，寄存器更少（EAX 而不是 RAX），指令更短。入门阶段用 32 位更容易理解。后面熟练了再切 64 位。

## 用 x64dbg 打开程序

打开 x64dbg 解压目录下的 **x32dbg.exe**（32 位版本，注意不是 x64dbg.exe，x64dbg.exe 是调试 64 位程序的）。把刚编译的 exe 直接**拖到 x32dbg 窗口上**。

<!-- 📸 截图：x32dbg.exe 在解压目录中的位置 -->

程序加载后会**自动暂停**。停下来的位置在 `ntdll.dll` 的系统代码里，不是你的程序——这是正常的，Windows 在启动程序之前要先执行一些系统初始化。你的程序代码和数据在加载时就已经全部映射到内存里了，只是 CPU 还没开始执行你的 main 函数。

<!-- 📸 截图：刚加载时停在 ntdll 的状态 -->

按 **Alt+F9**（执行到用户代码），让 CPU 跳过系统代码，跑到你的程序代码区域。你可能会看到类似 `jmp _mainCRTStartup` 的指令，这是 C 运行时的初始化代码。不用管它，直接搜索字符串就能跳到关键位置。

<!-- 📸 截图：按 Alt+F9 后停在 CRT 启动代码 -->

<!-- 🎨 画图：x64dbg 四窗口布局示意 -->

你会看到四个主要区域：

| 区域 | 内容 | 你需要关注的 |
|------|------|-------------|
| CPU 窗口 | 反汇编代码（左边地址，中间指令，右边注释） | 这里是核心 |
| 寄存器窗口 | CPU 的"变量"（EAX、EBX、ECX 等） | 观察值的变化 |
| 内存窗口 | 程序的内存数据 | 偶尔用 |
| 堆栈窗口 | 函数调用栈 | 后面章节讲 |

现在看不懂没关系，后面会逐个搞懂。

## 找到关键跳转

大多数验证程序的逻辑都是：**比较输入和正确密码 → 相等就成功，不等就失败**。这个"相等/不等"在汇编里就是一条跳转指令。

操作步骤：

1. 在 CPU 窗口的反汇编代码区域里**右键**
2. 弹出菜单中选择 **Search for** → **Current Module** → **All referenced text strings**
3. 在弹出的窗口里找到 **"Correct!"** 或 **"Wrong!"**，**双击**它

<!-- 📸 截图：右键 → Search for → Current Module → All referenced text strings -->

<!-- 📸 截图：字符串列表窗口，找到 "Correct!" 或 "Wrong!" -->

<!-- 📸 截图：双击跳转后看到的代码位置 -->

双击后，x32dbg 会跳到引用 "Correct!" 的那行代码（通常是 `push <..."Correct!"...>`），**高亮的那行就是跳转目标。你要往上看**（往上滚几行），找验证逻辑。

**你不需要读懂所有代码**，只需要在高亮行附近找这个固定模式：

```
push    <..."reverse2026"...>         ← ★ 正确密码明文写死在这里！
...（几行 push 和 call）...
test    eax,eax                       ← ★ 检查比较结果
jne     <某个地址>                     ← ★ 紧接着一条 jne（关键！）
push    <..."Correct!\n"...>          ← 你双击跳转到的那行
...（几行 push 和 call）...
jmp     <某个地址>                     ← 一条 jmp 跳过下面
push    <..."Wrong!\n"...>            ← Wrong 分支
```

看不懂中间的 push 和 call 没关系，只要在 "Correct!" 和 "Wrong!" 附近找到 **`test eax,eax` + `jne`** 这两行就够了。

<!-- 🎨 画图：标注搜索跳转后看到的代码结构，高亮 jne 是关键 -->

带 ★ 的三行就是破解的关键：

1. **`push <..."reverse2026"...>`** — 正确密码直接写死在程序里，明文！逆向时你一眼就能看到
2. **`test eax,eax`** — 检查 strcmp 的返回值，0 表示相等
3. **`jne <Wrong分支>`** — 不相等就跳到 Wrong 分支

你只需要盯住带 ★ 的这三行。

> 你实际看到的代码会比我写的更长，函数名可能显示为 `call consoleapplication1.xxxxx` 而不是 `call <strcmp>`。没关系，**位置和结构是一样的**——在 "Correct!" 和 "Wrong!" 附近找到 `test eax,eax` 和 `jne` 就行。

<!-- 🎨 画图：strcmp → test → jne 执行流程图，标出跳转方向 -->

## 改一个字节

找到带 ★ 的 `jne` 那行后，我们要让它"失效"——不管比较结果如何，都走成功的路。

**方法一：NOP 掉（推荐）**

把跳转指令替换成 NOP（No Operation，空操作）。程序走到这里就直接往下执行，不再跳转：

1. 点击选中 `jne` 那行
2. 按 **Ctrl+9**（或右键 → Binary → Fill with NOPs）

这样 x64dbg 会自动把 `jne` 的所有字节都填充为 `NOP`，一步到位。

<!-- 📸 截图：修改前的 jne 指令 -->

<!-- 📸 截图：按空格后弹出编辑框 -->

<!-- 📸 截图：修改后变成两条 nop -->

**方法二：取反跳转**

把 `jne`（不相等则跳）改成 `je`（相等则跳）。这样效果完全反过来了：密码正确反而显示 Wrong，密码错误反而显示 Correct。

改完后，按 **F9**（运行）。这时程序的控制台窗口会出现，在控制台里随便输入一个错误密码，按回车：

<!-- 📸 截图：随便输入 "abc" 后显示 "Correct!" -->

恭喜，你刚刚完成了人生第一次逆向。你输入了错误的密码，但程序告诉你"正确"。

## 你刚才做了什么

回顾一下整个过程：

1. **自己写了一个验证程序** — 你完全知道它的逻辑
2. **用 x32dbg 打开它** — 搜索字符串定位验证代码
3. **NOP 掉条件跳转** — 绕过了密码验证

这就是逆向最核心的操作：**找到关键的判断，然后绕过它**。

你可能会问：如果程序不是我自己写的，我不知道逻辑怎么办？答案是一样的——搜索字符串、找条件跳转、NOP 掉。**不管程序多复杂，验证逻辑最终都会走到一个条件跳转。** 找到它，你就赢了。

后面所有章节都是在这个基础上展开的。
