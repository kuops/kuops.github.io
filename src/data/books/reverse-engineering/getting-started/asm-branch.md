# 比较跳转与条件分支

## 本章目标

上一章学了 `add`、`sub`、`and`、`xor` 这些算术和逻辑指令。它们会"算"，但不会"判断"。算完的结果留在寄存器和标志位里，没有下文。

这一章你将学会：

- 两条比较指令：`cmp`（减法比较）、`test`（按位与测试）
- 无条件跳转 `jmp` 和条件跳转 `jcc` 系列（`je`/`jne`/`jg`/`jl` 等）
- **三步链条**：比较 → 填写 EFLAGS → 条件跳转读 EFLAGS
- 三个固定模式：if/else、检查零值、循环
- 用跟踪表完整跟踪 `cmp + jne` 和 `test + jne` 的执行过程

学完本章，你能从一段汇编里认出 if/else 和循环——这是逆向最常见的两种结构。

## 比较和跳转：核心链条

所有条件判断，在汇编层面都是同一个三步流程：

```
┌──────────────────────────────────────────────────────────┐
│  第1步：cmp / test（比较）                                │
│  本质是减法 / AND，不存结果，只更新 EFLAGS 里的标志位       │
└──────────────────────────┬───────────────────────────────┘
                           ▼
┌──────────────────────────────────────────────────────────┐
│  第2步：EFLAGS（快递单）                                  │
│  ZF、SF、CF、OF 等标志位记录了比较结果                      │
└──────────────────────────┬───────────────────────────────┘
                           ▼
┌──────────────────────────────────────────────────────────┐
│  第3步：jcc（条件跳转）                                   │
│  读取 EFLAGS，满足条件就跳，不满足就往下走                   │
└──────────────────────────────────────────────────────────┘
```

<!-- 🎨 画图：三步链条流程图，cmp/test → EFLAGS → jcc，用箭头和数据流标注 -->

打个比方：**cmp/test 是快递员填快递单，EFLAGS 是快递单本身，jcc 是收件人读快递单。**

快递员（cmp/test）填完单子（EFLAGS）就走了，不关心谁会来读。收件人（jcc）来读单子，根据单子上的信息决定去哪里取件。两个人互不认识，唯一的联系就是那张快递单（EFLAGS）。

这个三步链条是逆向工程**最核心的知识**。你在 x64dbg 里看到的绝大多数代码，本质上都是"比较 → 跳转"的反复组合。第一章你 NOP 掉的 `test eax, eax` + `jne`，就是这条链条的实例。

## cmp：比较

```
cmp eax, 100                ; 比较 eax 和 100
cmp eax, ebx                ; 比较 eax 和 ebx
cmp dword ptr [ebp-4], 0    ; 比较局部变量和 0
```

`cmp` 的本质是**减法**——计算 `目的操作数 - 源操作数`，但**不存储结果**，只根据差值更新 EFLAGS。

和 `sub` 的唯一区别：`sub` 把差值存回目的操作数，`cmp` 算完就扔，目的操作数不变。

**影响所有 6 个标志位**：ZF、SF、CF、OF、PF、AF。

### 三种情况的跟踪

假设 EAX = `100`（`0x64`），EIP = `00401000`：

**场景 1：相等（EAX == 100）**

```
指令          EAX       ZF  SF  CF  EIP       说明
──────────────────────────────────────────────────────
初始状态      00000064  0   0   0   00401000
cmp eax, 100  00000064  1   0   0   00401005  100-100=0，ZF=1，EAX 不变
```

差值为 0 → ZF=1（零标志置位）。这是 `cmp` 最常见的结果——**相等时 ZF=1**。

**场景 2：大于（EAX > 源操作数）**

```
指令          EAX       ZF  SF  CF  EIP       说明
──────────────────────────────────────────────────────
初始状态      00000064  0   0   0   00401000
cmp eax, 50   00000064  0   0   0   00401005  100-50=50，非零正数，无借位
```

差值 = 50，非零 → ZF=0；正数 → SF=0；没借位 → CF=0。三个关键标志位全为 0。

**场景 3：小于（EAX < 源操作数）**

```
指令          EAX       ZF  SF  CF  EIP       说明
──────────────────────────────────────────────────────
初始状态      00000064  0   0   0   00401000
cmp eax, 200  00000064  0   1   1   00401005  100-200 不够减，借位
```

无符号角度 100 减 200 不够减 → 借位 → CF=1。结果 `0xFFFFFF9C`（-100 的补码），最高位为 1 → SF=1。非零 → ZF=0。

<!-- 📸 截图：x64dbg 中执行三种 cmp 后 EFLAGS 窗口的变化 -->

### cmp 结果速查

| 比较                      | ZF  | SF  | CF  | 含义                      |
| ------------------------- | --- | --- | --- | ------------------------- |
| `cmp eax, 100`（eax=100） | 1   | 0   | 0   | 相等                      |
| `cmp eax, 50`（eax=100）  | 0   | 0   | 0   | eax 更大（无符号+有符号） |
| `cmp eax, 200`（eax=100） | 0   | 1   | 1   | eax 更小（无符号+有符号） |

**关键结论**：`cmp a, b` 之后——

- ZF=1 → a == b
- ZF=0 且 CF=0 → a > b（无符号）
- CF=1 → a < b（无符号）

## test：测试

```
test eax, eax               ; 检查 eax 是否为零
test eax, 0x80              ; 检查 eax 的第 7 位是否为 1
test al, 0x01               ; 检查 al 的最低位
```

`test` 的本质是**按位与（AND）**——计算 `目的操作数 & 源操作数`，但**不存储结果**，只根据 AND 的结果更新 EFLAGS。

和 `and` 的唯一区别：`and` 把结果存回目的操作数，`test` 算完就扔，目的操作数不变。

**影响标志位**：ZF/SF/PF 更新，**CF=0，OF=0**（和 `and` 一样，逻辑指令永远清零这两个）。

### 用法 1：检查零值

`test eax, eax` 是逆向中**出现频率最高**的指令之一。它用 eax 和自己做 AND——一个数和自己 AND，结果还是自己。所以这个指令的实际效果是：**根据 eax 的值更新标志位，但不改变 eax**。

```
EAX 值       AND 结果    ZF  SF  含义
────────────────────────────────────────
0x00000000   0x00000000  1   0   eax == 0
0x00000064   0x00000064  0   0   eax != 0，正数
0xFFFFFFFF   0xFFFFFFFF  0   1   eax != 0，负数（最高位为 1）
```

**eax 为零 → ZF=1，非零 → ZF=0**。后面紧跟 `je` 或 `jne`，就是判断 eax 是不是 0。

假设 EAX = `0x00000000`，EIP = `00401000`：

```
指令             EAX         ZF  SF  CF  EIP       说明
─────────────────────────────────────────────────────────
初始状态         00000000    0   0   0   00401000
test eax, eax    00000000    1   0   0   00401002  0&0=0，ZF=1
```

EAX = `0x00000005`：

```
指令             EAX         ZF  SF  CF  EIP       说明
─────────────────────────────────────────────────────────
初始状态         00000005    0   0   0   00401000
test eax, eax    00000005    0   0   0   00401002  5&5=5，非零，ZF=0
```

### 用法 2：检查特定位

```
test eax, 0x80              ; 检查第 7 位
```

假设 EAX = `0x000000AB`（二进制 `...1010 1011`），EIP = `00401000`：

```
指令          EAX         ZF  SF  CF  EIP       说明
─────────────────────────────────────────────────────────
初始状态      000000AB    0   0   0   00401000
test eax, 80  000000AB    0   0   0   00401005  0xAB & 0x80 = 0x80，非零
```

```
0x000000AB  =  ...1010 1011
0x00000080  =  ...1000 0000
─────────────────────────────
AND 结果    =  ...1000 0000  =  0x80（非零，第 7 位是 1）
```

假设 EAX = `0x0000007F`（二进制 `...0111 1111`，第 7 位为 0）：

```
指令          EAX         ZF  SF  CF  EIP       说明
─────────────────────────────────────────────────────────
初始状态      0000007F    0   0   0   00401000
test eax, 80  0000007F    1   0   0   00401005  0x7F & 0x80 = 0x00，ZF=1
```

```
0x0000007F  =  ...0111 1111
0x00000080  =  ...1000 0000
─────────────────────────────
AND 结果    =  ...0000 0000  =  0x00（第 7 位是 0，结果为零）
```

这种模式在权限检查中常见——用一个掩码检查某个标志位是否被设置。

## jmp：无条件跳转

```
jmp 00402000                ; 直接跳到 00402000
```

`jmp` 类似 C 的 `goto`——无条件、不判断，直接把 EIP 设为目标地址。没有条件，不看 EFLAGS。

在 if/else 结构里，`jmp` 经常出现在 if 体的末尾，用来**跳过 else 体**（后面"固定模式"部分会看到）。

`jmp` **不影响任何标志位**。

## 条件跳转：jcc 系列

条件跳转指令读取 EFLAGS 的标志位，**满足条件就跳转，不满足就往下走**（EIP 正常前进到下一条）。它们本身不改任何标志位，也不改任何寄存器——只读快递单，不改快递单。

x86 有几十个条件跳转指令，但**日常逆向只用到 10 个左右**。下面按"使用频率"排列：

| 助记符  | 跳转条件      | 中文含义       | 典型场景                   |
| ------- | ------------- | -------------- | -------------------------- |
| **je**  | ZF=1          | 相等则跳       | `cmp` 后判断相等           |
| **jne** | ZF=0          | 不等则跳       | `cmp` 后判断不等           |
| **jg**  | ZF=0 且 SF=OF | 有符号大于则跳 | `cmp a,b` 后 a>b（有符号） |
| **jge** | SF=OF         | 有符号≥则跳    | `cmp a,b` 后 a≥b（有符号） |
| **jl**  | SF≠OF         | 有符号小于则跳 | `cmp a,b` 后 a<b（有符号） |
| **jle** | ZF=1 或 SF≠OF | 有符号≤则跳    | `cmp a,b` 后 a≤b（有符号） |
| **ja**  | CF=0 且 ZF=0  | 无符号大于则跳 | `cmp a,b` 后 a>b（无符号） |
| **jae** | CF=0          | 无符号≥则跳    | `cmp a,b` 后 a≥b（无符号） |
| **jb**  | CF=1          | 无符号小于则跳 | `cmp a,b` 后 a<b（无符号） |
| **jbe** | CF=1 或 ZF=1  | 无符号≤则跳    | `cmp a,b` 后 a≤b（无符号） |

> 助记符里的 `e` = equal（等于），`n` = not（不），`g` = greater（大于），`l` = less（小于），`a` = above（无符号大于），`b` = below（无符号小于）。

**逆向实战中 `je`/`jne` 占 80% 以上的场景**。大多数判断都是"相等/不等"，很少需要区分大于/小于。初学阶段把精力放在 `je`/`jne` 上，遇到其他指令再查表。

### 简易翻译法

面对一条 `jcc` 指令，怎么快速理解它在判断什么？一个简单的方法：

**把助记符翻译成运算符，放在 cmp 的两个操作数之间。**

```
cmp  eax, ebx
jcc  <目标>
```

| jcc     | → 运算符 | 心理翻译                    |
| ------- | -------- | --------------------------- |
| **je**  | `==`     | `if (eax == ebx) goto 目标` |
| **jne** | `!=`     | `if (eax != ebx) goto 目标` |
| **jg**  | `>`      | `if (eax > ebx) goto 目标`  |
| **jge** | `>=`     | `if (eax >= ebx) goto 目标` |
| **jl**  | `<`      | `if (eax < ebx) goto 目标`  |
| **jle** | `<=`     | `if (eax <= ebx) goto 目标` |

这个方法只适用于 `cmp` 正上方的 `jcc`。如果你看到的是 `test`，翻译方式不同（后面讲）。

举个例子：

```asm
cmp  dword ptr [ebp-8], 100
jne  00401050
```

心里翻译：`if ([ebp-8] != 100) goto 00401050`。局部变量不等于 100 就跳走。

再一个：

```asm
cmp  eax, ecx
jle  00401030
```

心里翻译：`if (eax <= ecx) goto 00401030`。eax 小于等于 ecx 就跳走。

## 完整跟踪：cmp + jne

三步链条的完整过程。以下面这段代码为例：

```asm
00401000  mov  eax, 100
00401005  mov  ebx, 100
0040100A  cmp  eax, ebx
0040100C  jne  00401020
00401012  mov  ecx, 1
00401017  jmp  00401025
00401020  mov  ecx, 0
00401025  ...
```

这段代码的逻辑：如果 EAX == EBX，ECX = 1；否则 ECX = 0。

### 场景 1：eax == ebx（EAX = 100, EBX = 100）

```
指令            EAX       EBX       ECX  ZF  EIP       说明
────────────────────────────────────────────────────────────────────
初始状态        00000064  00000064  ??   0   00401000
mov eax, 100    00000064  00000064  ??   0   00401005  eax=100(0x64)
mov ebx, 100    00000064  00000064  ??   0   0040100A  ebx=100(0x64)
cmp eax, ebx    00000064  00000064  ??   1   0040100C  100-100=0，ZF=1
jne 00401020    00000064  00000064  ??   1   00401012  ZF=1，不跳！往下走
mov ecx, 1      00000064  00000064  1    1   00401017  ecx=1
jmp 00401025    00000064  00000064  1    1   00401025  无条件跳走，跳过 else
```

EAX == EBX → `cmp` 算出差为 0 → ZF=1 → `jne` 要求 ZF=0 才跳 → **不跳** → 执行 `mov ecx, 1` → `jmp` 跳过 else 分支。最终 ECX = 1。

### 场景 2：eax != ebx（EAX = 100, EBX = 50）

```
指令            EAX       EBX       ECX  ZF  EIP       说明
────────────────────────────────────────────────────────────────────
初始状态        00000064  00000032  ??   0   00401000
mov eax, 100    00000064  00000032  ??   0   00401005  eax=100(0x64)
mov ebx, 100    00000064  00000064  ??   0   0040100A  ebx=100(0x64)
```

等等，让我改一下——假设初始 EBX = 50（`0x32`），然后 `mov ebx, 100` 改成 `mov ebx, 50`：

```asm
00401000  mov  eax, 100
00401005  mov  ebx, 50
0040100A  cmp  eax, ebx
0040100C  jne  00401020
00401012  mov  ecx, 1
00401017  jmp  00401025
00401020  mov  ecx, 0
00401025  ...
```

```
指令            EAX       EBX       ECX  ZF  EIP       说明
────────────────────────────────────────────────────────────────────
初始状态        00000000  00000000  ??   0   00401000
mov eax, 100    00000064  00000000  ??   0   00401005  eax=100
mov ebx, 50     00000064  00000032  ??   0   0040100A  ebx=50
cmp eax, ebx    00000064  00000032  ??   0   0040100C  100-50=50，非零，ZF=0
jne 00401020    00000064  00000032  ??   0   00401020  ZF=0，跳！EIP=00401020
mov ecx, 0      00000064  00000032  0    0   00401025  ecx=0
```

EAX != EBX → `cmp` 算出差为 50（非零）→ ZF=0 → `jne` 要求 ZF=0 → **跳！** → 跳到 `00401020`，执行 `mov ecx, 0`。最终 ECX = 0。

注意 `jne` 跳转后，`mov ecx, 1` 和 `jmp 00401025` 都被跳过了，直接执行 `mov ecx, 0`。

## 完整跟踪：test + jne

回到第一章 first-crack 的代码。`strcmp` 比较两个字符串，相等返回 0，不等返回非零值。返回值在 EAX 里。后面的代码：

```asm
test eax, eax
jne  <Wrong分支>
```

### 场景 1：密码正确（strcmp 返回 0）

```
指令              EAX         ZF  EIP       说明
──────────────────────────────────────────────────────
(strcmp 返回)     00000000    0   00401020  strcmp 返回 0（相等）
test eax, eax     00000000    1   00401022  0&0=0，ZF=1
jne  <Wrong>      00000000    1   00401028  ZF=1，不跳！往下走 → Correct!
```

`test eax, eax` 发现 EAX = 0 → ZF=1 → `jne` 要求 ZF=0 才跳 → 不跳 → 继续执行 Correct 分支。

### 场景 2：密码错误（strcmp 返回 1）

```
指令              EAX         ZF  EIP       说明
──────────────────────────────────────────────────────
(strcmp 返回)     00000001    0   00401020  strcmp 返回 1（不等）
test eax, eax     00000001    0   00401022  1&1=1，非零，ZF=0
jne  <Wrong>      00000001    0   00401050  ZF=0，跳！→ Wrong!
```

`test eax, eax` 发现 EAX = 1（非零）→ ZF=0 → `jne` 要求 ZF=0 → **跳！** → 跳到 Wrong 分支。

现在你完全理解第一章那个 `test + jne` 了。**NOP 掉 `jne` 后，不管 ZF 是什么值，CPU 都不会跳，永远走 Correct 分支。**

## 固定模式：if/else

C 代码：

```c
if (eax == 100) {
    ecx = 1;
} else {
    ecx = 0;
}
```

编译后的汇编：

```asm
cmp  eax, 100
jne  else_branch
mov  ecx, 1
jmp  end
else_branch:
mov  ecx, 0
end:
```

<!-- 🎨 画图：if/else 的汇编结构流程图，标注 cmp → jne → if体 → jmp → else体 -->

### 关键心法

**`jcc` 的跳转方向和 C 的 if 条件相反。**

- C 写的是 `if (eax == 100)`，翻译成"如果等于 100 就执行 if 体"
- 汇编写的是 `jne else_branch`，翻译成"如果不等于 100 就跳到 else"

为什么？因为 CPU 的设计思路是"**不满足条件就跳走**"。满足条件时，不跳，直接往下执行 if 体。不满足时，跳走，去 else 体。

**记忆方法**：看到 `jcc`，把它的条件**取反**，就是 if 的条件。

| 汇编         | jcc 条件 | 取反 → C 的 if 条件 |
| ------------ | -------- | ------------------- |
| `je` (跳走)  | ==       | **!=**              |
| `jne` (跳走) | !=       | **==**              |
| `jg` (跳走)  | >        | **<=**              |
| `jl` (跳走)  | <        | **>=**              |
| `jge` (跳走) | >=       | **<**               |
| `jle` (跳走) | <=       | **>**               |

也就是说，`jcc` 后面紧跟的代码块就是 if 体（满足条件时执行的部分），`jcc` 跳转目标是 else 体（不满足时执行的部分）。

### 更多 if/else 变体

**有符号比较：**

```c
if (eax > ebx) { ... } else { ... }
```

```asm
cmp  eax, ebx
jle  else_branch    ; jle = 有符号 ≤，取反得到 >
mov  ecx, 1
jmp  end
else_branch:
mov  ecx, 0
end:
```

**无符号比较：**

```c
if (eax >= ebx) { ... } else { ... }
```

```asm
cmp  eax, ebx
jb   else_branch    ; jb = 无符号 <，取反得到 ≥
mov  ecx, 1
jmp  end
else_branch:
mov  ecx, 0
end:
```

**没有 else 的情况：**

```c
if (eax == 0) {
    eax = 1;
}
```

```asm
cmp  eax, 0
jne  skip
mov  eax, 1
skip:
```

没有 else 时，`jcc` 跳过的就是整个 if 体，末尾不需要 `jmp`。

## 固定模式：检查零值

`test eax, eax` + `jne`/`je` 是另一种高频模式。它等价于 C 的 `if (eax == 0)` 或 `if (eax != 0)`。

```c
if (eax == 0) {
    eax = -1;
}
```

```asm
test eax, eax
jne  skip          ; 非零就跳走（取反：if eax == 0 才往下走）
or   eax, 0xFFFFFFFF   ; eax = -1
skip:
```

为什么不用 `cmp eax, 0`？因为 `test eax, eax` 机器码更短（2 字节 vs 5 字节），编译器会优先选 `test`。

| C 代码        | 汇编                | 说明                      |
| ------------- | ------------------- | ------------------------- |
| `if (x == 0)` | `test x, x` + `jne` | 非零跳走 = 等于零才往下走 |
| `if (x != 0)` | `test x, x` + `je`  | 等于零跳走 = 非零才往下走 |
| `if (!x)`     | `test x, x` + `jne` | 同 `x == 0`               |
| `if (x)`      | `test x, x` + `je`  | 同 `x != 0`               |

**逆向时看到 `test reg, reg` + `jne`/`je`，就是在检查那个寄存器是不是 0。**

## 固定模式：循环

### 计数循环：dec + jne

```c
for (int i = 3; i > 0; i--) {
    eax += eax;
}
```

编译后：

```asm
mov  ecx, 3
loop_start:
add  eax, eax
dec  ecx
jne  loop_start
```

<!-- 🎨 画图：计数循环流程图——ecx=3 → add → dec → jne(回到add) → ... → dec → ecx=0 → jne不跳 -->

这个循环的结构：**循环体 + 计数器递减 + 条件跳回**。`dec ecx` 把 ECX 减 1，`jne` 检查 ECX 是否为零——不为零就跳回循环体开头。

**完整跟踪**（假设初始 EAX = `0x00000001`）：

```
指令              EAX       ECX       ZF  EIP       说明
─────────────────────────────────────────────────────────────────
mov ecx, 3        00000001  00000003  0   00401000  循环 3 次
                                  第 1 次循环
add eax, eax      00000002  00000003  0   00401002  1+1=2
dec ecx           00000002  00000002  0   00401003  3-1=2，非零，ZF=0
jne loop_start    00000002  00000002  0   00401002  ZF=0，跳回！
                                  第 2 次循环
add eax, eax      00000004  00000002  0   00401002  2+2=4
dec ecx           00000004  00000001  0   00401003  2-1=1，非零，ZF=0
jne loop_start    00000004  00000001  0   00401002  ZF=0，跳回！
                                  第 3 次循环
add eax, eax      00000008  00000001  0   00401002  4+4=8
dec ecx           00000008  00000000  1   00401003  1-1=0，ZF=1
jne loop_start    00000008  00000000  1   00401005  ZF=1，不跳！循环结束
```

最终 EAX = `0x00000008`（1 → 2 → 4 → 8，翻倍 3 次）。ECX = `0x00000000`（从 3 减到 0）。

注意 `dec` 不影响 CF（上一章讲过），但它会影响 ZF——`dec ecx` 把 ECX 减到 0 时 ZF=1，`jne` 读到 ZF=1 就不跳了，循环结束。

### 条件循环：jmp 回跳 + 条件跳出

```c
while (eax < 100) {
    eax = eax + 10;
}
```

编译后：

```asm
loop_start:
cmp  eax, 100
jge  loop_end          ; eax >= 100 就跳出
add  eax, 10
jmp  loop_start        ; 无条件跳回开头
loop_end:
```

这个结构是：**循环开头检查条件 → 不满足就跳走（跳出循环） → 循环体 → 无条件跳回开头**。

**完整跟踪**（假设初始 EAX = `0x00000050`，即 80）：

```
指令              EAX       ZF  SF  EIP       说明
──────────────────────────────────────────────────────────
                                第 1 次循环
cmp eax, 100      00000050  0   0   00401003  80-100<0，但 SF≠OF
jge loop_end      00000050  0   0   00401005  不满足 ≥，不跳
add eax, 10       0000005A  0   0   00401008  80+10=90(0x5A)
jmp loop_start    0000005A  0   0   00401000  无条件跳回
                                第 2 次循环
cmp eax, 100      0000005A  0   0   00401003  90-100<0
jge loop_end      0000005A  0   0   00401005  不满足 ≥，不跳
add eax, 10       00000064  0   0   00401008  90+10=100(0x64)
jmp loop_start    00000064  0   0   00401000  无条件跳回
                                第 3 次循环
cmp eax, 100      00000064  1   0   00401003  100-100=0，ZF=1
jge loop_end      00000064  1   0   00401020  SF=OF=0，满足 ≥，跳出！
```

最终 EAX = `0x00000064`（100）。循环执行了 2 次（80→90→100），第 3 次检查时 `cmp` 发现 EAX == 100，`jge`（≥）满足条件，跳出循环。

### 循环模式速查

| C 循环                  | 汇编模式                     | 特征                       |
| ----------------------- | ---------------------------- | -------------------------- |
| `for/while`（向下计数） | `dec ecx` + `jne loop_start` | ECX 递减到 0 停止          |
| `while`（条件循环）     | `jmp 回开头` + `条件跳出`    | 循环末尾 `jmp`，开头 `jcc` |
| `do-while`              | `循环体` + `jcc 回开头`      | 至少执行一次，条件在末尾   |

## 指令对标志位影响总结表

本章新学的指令全部加入总结表：

| 指令       | ZF   | SF   | CF       | OF       | 说明               |
| ---------- | ---- | ---- | -------- | -------- | ------------------ |
| **cmp**    | 更新 | 更新 | 更新     | 更新     | 减法比较，不存结果 |
| **test**   | 更新 | 更新 | **清零** | **清零** | AND 测试，不存结果 |
| **jmp**    | 不变 | 不变 | 不变     | 不变     | 无条件跳转         |
| **je/jne** | 不变 | 不变 | 不变     | 不变     | 读 ZF              |
| **jg/jle** | 不变 | 不变 | 不变     | 不变     | 读 ZF、SF、OF      |
| **jl/jge** | 不变 | 不变 | 不变     | 不变     | 读 SF、OF          |
| **ja/jbe** | 不变 | 不变 | 不变     | 不变     | 读 CF、ZF          |
| **jb/jae** | 不变 | 不变 | 不变     | 不变     | 读 CF              |

**所有跳转指令（jmp 和 jcc）都不改标志位。** 它们只读标志位，不写。`cmp`/`test` 只写标志位，不存运算结果。两者通过 EFLAGS 这张"快递单"配合工作。

再加上前两章的指令：

| 指令   | ZF   | SF   | CF       | OF       | 说明                   |
| ------ | ---- | ---- | -------- | -------- | ---------------------- |
| `mov`  | 不变 | 不变 | 不变     | 不变     | 搬数据，不改任何标志位 |
| `lea`  | 不变 | 不变 | 不变     | 不变     | 算地址，不改任何标志位 |
| `nop`  | 不变 | 不变 | 不变     | 不变     | 空操作                 |
| `add`  | 更新 | 更新 | 更新     | 更新     | 加法                   |
| `sub`  | 更新 | 更新 | 更新     | 更新     | 减法                   |
| `inc`  | 更新 | 更新 | **不变** | 更新     | +1                     |
| `dec`  | 更新 | 更新 | **不变** | 更新     | -1                     |
| `and`  | 更新 | 更新 | **清零** | **清零** | 按位与                 |
| `or`   | 更新 | 更新 | **清零** | **清零** | 按位或                 |
| `xor`  | 更新 | 更新 | **清零** | **清零** | 按位异或               |
| `not`  | 不变 | 不变 | 不变     | 不变     | 按位取反               |
| `cmp`  | 更新 | 更新 | 更新     | 更新     | 减法比较（本章）       |
| `test` | 更新 | 更新 | **清零** | **清零** | AND 测试（本章）       |

**记忆规律**：

- **不改标志位**：mov、lea、nop、not、所有跳转指令
- **清零 CF 和 OF**：and、or、xor、test（逻辑指令家族）
- **不改 CF**：inc、dec（保护循环中的进位）
- **全部更新**：add、sub、cmp（算术指令家族）

## 练习

### 第一题：预测标志位

初始 EAX = `0x0000000A`（10），EIP = `00401000`。执行以下指令后，ZF、SF、CF 各是什么？

```
cmp eax, 10
```

<details>
<summary>答案</summary>

ZF=1，SF=0，CF=0。

10 - 10 = 0 → ZF=1（相等）。没有借位，没有负数。

</details>

### 第二题：预测标志位

初始 EAX = `0x00000005`（5），EIP = `00401000`。执行以下指令后，ZF、SF、CF 各是什么？

```
cmp eax, 10
```

<details>
<summary>答案</summary>

ZF=0，SF=1，CF=1。

5 - 10：不够减，借位 → CF=1。结果是 `0xFFFFFFFB`（-5 的补码），最高位 1 → SF=1。非零 → ZF=0。

</details>

### 第三题：还原 if/else

阅读以下汇编，还原等价的 C 代码：

```asm
cmp  dword ptr [ebp-4], 0
jle  00401030
mov  eax, 1
jmp  00401035
00401030:
mov  eax, 0
00401035:
```

<details>
<summary>答案</summary>

```c
if (local_var > 0) {
    eax = 1;
} else {
    eax = 0;
}
```

`jle`（有符号 ≤）跳走 → 取反得到 `>`。所以条件是 `[ebp-4] > 0`。

</details>

### 第四题：还原循环

阅读以下汇编，还原等价的 C 循环：

```asm
00401000  mov  ecx, 5
00401005  xor  eax, eax
00401007  add  eax, ecx
00401009  dec  ecx
0040100A  jne  00401007
```

这段代码计算的是什么？

<details>
<summary>答案</summary>

```c
int ecx = 5;
int eax = 0;
while (ecx != 0) {
    eax += ecx;
    ecx--;
}
```

计算的是 5 + 4 + 3 + 2 + 1 = **15**（`0x0F`）。

跟踪：

```

xor eax, eax      00000000  00000005  1   00401007
add eax, ecx      00000005  00000005  0   00401009  eax=0+5=5
dec ecx           00000005  00000004  0   0040100A  ecx=4，ZF=0
jne 00401007      00000005  00000004  0   00401007  跳回
add eax, ecx      00000009  00000004  0   00401009  eax=5+4=9
dec ecx           00000009  00000003  0   0040100A  ecx=3
jne 00401007      00000009  00000003  0   00401007  跳回
add eax, ecx      0000000C  00000003  0   00401009  eax=9+3=12
dec ecx           0000000C  00000002  0   0040100A  ecx=2
```

### 第五题：test + je

初始 EAX = `0x00000000`。执行以下指令后，EIP 跳到哪里？

```asm
00401000  test eax, eax
00401002  je   00401050
00401008  mov  ecx, 1
00401010  ...
00401050  mov  ecx, 0
```

<details>
<summary>答案</summary>

EIP = `00401050`。

`test eax, eax`：EAX = 0，0 AND 0 = 0 → ZF=1。`je` 要求 ZF=1 → 跳转到 `00401050`。ECX 被赋值为 0。

</details>

### 第六题：还原 while 循环

阅读以下汇编，还原等价的 C 代码。假设 `[ebp-4]` 是一个局部变量 `x`：

```asm
00401000  cmp  dword ptr [ebp-4], 50
00401007  jge  00401020
00401009  add  dword ptr [ebp-4], 10
00401010  jmp  00401000
00401020  ...
```

<details>
<summary>答案</summary>

```c
while (x < 50) {
    x += 10;
}
```

`jge`（≥）跳出 → 条件取反是 `<`。所以是 `while (x < 50)`。每次加 10。

</details>

### 第七题：x64dbg 验证

打开第一章的 CrackMe 程序，找到 `test eax, eax` + `jne` 那个位置。做以下操作：

1. 在 `jne` 那行设断点（F2），按 F9 运行到那里
2. 在寄存器窗口双击 EAX，手动改成 `0x00000000`
3. 按 F8 单步执行 `jne`——它跳了吗？为什么？
4. 重新运行，这次把 EAX 改成 `0x00000001`
5. 按 F8 单步执行 `jne`——它跳了吗？为什么？

<details>
<summary>答案</summary>

1. EAX = 0 → `test eax, eax` 使 ZF=1 → `jne` 要求 ZF=0 → **不跳**，走 Correct 分支
2. EAX = 1 → `test eax, eax` 使 ZF=0 → `jne` 要求 ZF=0 → **跳**，走 Wrong 分支

这就是手动改标志位来控制程序行为。和 NOP 掉 `jne` 效果类似，但 NOP 是永久修改（改了代码），手动改寄存器是临时修改（下次运行恢复）。

</details>

### 第八题：jne 改 je

在第一章的 CrackMe 程序里，把 `jne` 改成 `je`（不是 NOP 掉，是改指令）。回答以下问题：

1. 输入正确密码 "reverse2026"，程序输出什么？
2. 输入错误密码 "abc"，程序输出什么？
3. 为什么？

提示：`jne` 和 `je` 的机器码只有 1 字节之差——`jne` 是 `75`，`je` 是 `74`。在 x64dbg 里选中 `jne` 那行，按空格键，把 `jne` 改成 `je` 即可。

<details>
<summary>答案</summary>

1. 输入正确密码 "reverse2026" → 输出 **Wrong**
2. 输入错误密码 "abc" → 输出 **Correct**
3. 因为跳转条件被取反了。原来 `jne`（不等就跳到 Wrong），改成 `je` 后变成（相等就跳到 Wrong）。所以密码正确时反而跳到 Wrong，密码错误时不跳，走到 Correct。

</details>
