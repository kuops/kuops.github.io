# 数字在计算机里长什么样

## 动手目标

今天结束你会：

1. 搞懂二进制、十六进制和十进制之间的转换
2. 理解内存地址、字节序这些概念
3. 用 010 Editor 打开一个 exe，看到它的真实二进制内容

## 为什么到处是 0x

你打开 x64dbg 之后，满眼都是 `0x00401000`、`0x0A`、`0xFF` 这样的数字。`0x` 前缀表示这是**十六进制**（Hexadecimal）。

为什么用十六进制？因为二进制太长了：

| 十进制 | 二进制             | 十六进制 |
| ------ | ------------------ | -------- |
| 10     | `00001010`         | `0x0A`   |
| 255    | `11111111`         | `0xFF`   |
| 65535  | `1111111111111111` | `0xFFFF` |

每 4 个二进制位刚好对应 1 个十六进制位。一个字节（8 位）用两个十六进制数字就能表示，比如 `0xFF` = 255 = `11111111`。

<!-- 🎨 画图：二进制/十进制/十六进制对照表，重点是 0-F 对应关系 -->

**你不需要心算转换**。Windows 自带计算器切换到"程序员模式"就能转换，x64dbg 里寄存器窗口右键也可以切换显示格式。

<!-- 📸 截图：Windows 计算器程序员模式，HEX/DEC/OCT/BIN 切换 -->

## 内存模型

程序运行时，操作系统给它分配一段**虚拟地址空间**。32 位程序的地址空间是 0x00000000 到 0xFFFFFFFF（4GB）。

但不是所有地址都能用。一个典型进程的内存布局是这样的：

<!-- 🎨 画图：进程内存布局（从低地址到高地址：代码段 .text → 数据段 .data/.rdata → 堆 heap → ... → 栈 stack → 内核空间） -->

| 区域        | 内容                                       |
| ----------- | ------------------------------------------ |
| .text       | 程序的机器码（你看到的汇编指令就存在这里） |
| .data       | 已初始化的全局变量                         |
| .rdata      | 只读数据（字符串常量等）                   |
| 堆（Heap）  | 动态分配的内存（malloc/new）               |
| 栈（Stack） | 局部变量、函数调用信息                     |

你在 x64dbg 里看到的地址，就是这块空间里的位置。比如 `0x00401000` 通常在 .text 段，是程序的代码。

## 字节序：大小端

一个 4 字节的整数 `0x12345678`，在内存里怎么存？

<!-- 🎨 画图：大端序 vs 小端序的存储对比 -->

**小端序（Little-Endian）** — 低位字节放在低地址（x86/x64 采用）：

```
地址:     0x00  0x01  0x02  0x03
内容:     0x78  0x56  0x34  0x12
```

**大端序（Big-Endian）** — 高位字节放在低地址（网络传输用）：

```
地址:     0x00  0x01  0x02  0x03
内容:     0x12  0x34  0x56  0x78
```

逆向中 99% 的情况是小端序。在 x64dbg 的内存窗口里看数据时要注意：你看到的字节顺序是反的。

<!-- 📸 截图：x64dbg 内存窗口，选中一个 DWORD 值，标注小端序的字节顺序 -->

## 用 010 Editor 看二进制文件

010 Editor 是一个专业的十六进制编辑器，比普通记事本强在哪？它能按**模板**解析二进制文件的结构。

<!-- 📸 截图：010 Editor 打开一个 exe 的界面，左侧十六进制，右侧 ASCII -->

操作步骤：

1. 打开 010 Editor，把一个 exe 拖进去
2. 点击 **Templates** → **PE Template** → **Run**
3. 左边会出现解析后的结构树

<!-- 📸 截图：010 Editor 运行 PE 模板后的结果，展开结构树 -->

你会看到：

- **DOS Header** — 每个 exe 开头都有，包含一个指向 PE 头的偏移
- **PE Header** — 程序的元信息（目标平台、时间戳等）
- **Section Table** — 各个节（.text、.data 等）的信息
- **Sections** — 实际的代码和数据

现在看不懂没关系，后面第 16 章会详细讲 PE 结构。这里你只需要建立一个感觉：**程序就是一堆二进制数据，按照特定格式组织在一起**。

<!-- 📸 截图：010 Editor 中选中 "This program cannot be run in DOS mode" 字符串 -->

试着在十六进制数据里滚动，你能看到一些明文字符串——这就是程序里写死的文字。

## ASCII 与字符串

计算机用数字表示字符。ASCII 编码中：

- `0x41` = 'A'
- `0x61` = 'a'
- `0x30` = '0'
- `0x00` = 字符串结束符（\0）

在 x64dbg 里搜索字符串，本质就是在内存里找一段连续的 ASCII 字节，以 `0x00` 结尾。

<!-- 📸 截图：010 Editor 中选中一段 ASCII 文本，下方显示对应的十六进制值 -->

## 寄存器速查

上一章你看到了寄存器窗口，这里做一个快速介绍。寄存器是 CPU 内部的高速存储单元，可以理解为 CPU 自带的"变量"。

<!-- 🎨 画图：x86 通用寄存器及其用途 -->

| 寄存器 | 常见用途                     |
| ------ | ---------------------------- |
| EAX    | 函数返回值、乘除法           |
| EBX    | 通用（基址）                 |
| ECX    | 计数器、this 指针（C++）     |
| EDX    | 乘除法辅助、I/O              |
| ESI    | 源索引（字符串操作）         |
| EDI    | 目标索引（字符串操作）       |
| ESP    | 栈指针（指向栈顶）           |
| EBP    | 帧指针（指向当前栈帧底部）   |
| EIP    | 指令指针（当前执行到哪里了） |

**最重要**的是 EIP（当前执行位置）、ESP 和 EBP（栈相关）、EAX（返回值）。其他的用到时再记。

## 练习

1. 用 Windows 计算器（程序员模式）做几个转换：`0x1F` = ?（十进制），255 = ?（十六进制），`0b10101010` = ?（十六进制）
2. 用 010 Editor 打开上一章的 CrackMe，滚动找到里面的字符串（比如 "Correct"、"Wrong"），记下它们的偏移地址
3. 对比 x64dbg 里看到的地址和 010 Editor 里的偏移，它们一样吗？为什么？

<details>
<summary>提示</summary>

x64dbg 里看到的是**加载到内存后的地址**（虚拟地址），010 Editor 里看到的是**文件中的偏移**（文件偏移）。它们不一样，因为 PE 文件加载到内存时节会被映射到不同的位置。后面讲 PE 结构时会详细解释。
