## 什么是内容集合

内容集合（Content Collections）是 Astro 内置的内容管理系统，用于组织、验证和查询 Markdown/MDX 内容。

## 定义集合

在 `src/content.config.ts` 中定义：

```ts
import { defineCollection, z } from 'astro:content';
import { glob } from 'astro/loaders';

const blog = defineCollection({
  loader: glob({ pattern: '**/*.md', base: './src/data/blog' }),
  schema: z.object({
    title: z.string(),
    description: z.string(),
    pubDatetime: z.date(),
    tags: z.array(z.string()),
  }),
});

export const collections = { blog };
```

### Schema 验证

使用 Zod 定义 frontmatter 的类型和校验规则：

```ts
schema: z.object({
  title: z.string(),
  description: z.string(),
  pubDatetime: z.date(),
  modDatetime: z.date().optional(),
  draft: z.boolean().default(false),
  tags: z.array(z.string()).default(['others']),
  image: z.object({
    src: z.string(),
    alt: z.string(),
  }).optional(),
}),
```

## 查询内容

在页面中获取集合数据：

```astro
---
import { getCollection, render } from 'astro:content';

const posts = await getCollection('blog', ({ data }) => {
  return !data.draft;
});

for (const post of posts) {
  const { Content } = await render(post);
}
---
```

### 常用查询模式

```ts
// 获取所有非草稿文章
const published = await getCollection('blog', ({ data }) => !data.draft);

// 按标签筛选
const tagged = await getCollection('blog', ({ data }) =>
  data.tags.includes('astro')
);

// 按日期排序
const sorted = published.sort(
  (a, b) => b.data.pubDatetime.getTime() - a.data.pubDatetime.getTime()
);
```

## 渲染内容

```astro
---
const { Content } = await render(post);
---

<article>
  <h1>{post.data.title}</h1>
  <Content />
</article>
```

## 最佳实践

1. **类型安全** — 始终在 schema 中定义所有字段
2. **默认值** — 为可选字段设置合理的默认值
3. **草稿机制** — 使用 `draft` 字段控制发布状态
4. **目录结构** — 按内容类型分目录存放
