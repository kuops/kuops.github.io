# 《逆向工程实战入门》设计文档

## 定位

- **目标读者**：计算机专业学生，会 C/C++，但没做过逆向
- **核心原则**：先爽再懂，每章直接开干，概念在实战中自然出现
- **发布形式**：博客连载，放在 books 系列中
- **MVP 范围**：10 章，写完即发布，后续可扩展开篇进阶

## 教学理念

### 三步循环法

每一轮学习都是：**直接做 → 做成了 → 搞懂刚才发生了什么**

1. 照着做，不解释原理，先成功再说
2. 成功后回头看，用自己写的代码对照理解
3. 练习：给汇编还原 C，或者给新程序自己分析

### 模式识别教学法

不教"汇编语言"，教"这段汇编对应什么 C 结构"：

| C 结构 | 汇编长相 | 记忆点 |
|--------|---------|--------|
| 变量赋值 | `mov [ebp-X], 立即数` | 往内存写东西 |
| if 判断 | `cmp` + `jxx` | 先比再跳 |
| 循环 | `cmp` + `jxx` + 末尾 `jmp` 回去 | 往回跳的就是循环 |
| 函数调用 | `push` 参数 → `call` → `add esp` | 压参、调用、清理 |
| 返回值 | 结果在 `eax` | eax 就是 return |
| 数组访问 | `基址 + 索引 * sizeof` | 连续内存 |
| 成员访问 | `[ecx + 偏移]` | ecx 是 this |
| 虚函数调用 | `call [eax + 偏移]` | 间接调用 = 虚表 |

### 知识点在实战中引入

不提前讲知识点，而是在"卡住"时插入：

- 第 1 轮爆破 CrackMe → 只需认识 CMP、JNE 两条指令
- 第 2 轮对照自己代码 → 自然学到 MOV、ADD、CALL、RET
- 第 4 轮追注册码 → 自然遇到 XOR、SHL、LEA（加密常见）
- 第 5 轮 IDA 看程序 → 自然认识虚表、this 指针形态

## 精讲要求

### 第 2-3 章 C↔汇编对照精讲

每个知识点走三步：
1. **看**：给 C 代码 + 对应汇编 + 逐行注释解释
2. **改**：改 C 代码（如 if 变 switch），让读者猜汇编怎么变，然后验证
3. **练**：给一段汇编，要求还原出 C 代码

必须精讲的知识点清单：

**第 2 章（上）：**
- 局部变量与 `[ebp-X]` 的关系
- 赋值、算术运算（add/sub/imul/idiv）
- 编译器优化识别（乘 2 用 shl、除以常量用乘法+移位）
- if/else → cmp + jxx，跳转方向的理解（jle 跳的是 else 不是 if）
- Release 优化：setcc、cmov 等变形也要认识

**第 3 章（下）：**
- switch：少量 case = 连续 cmp+je，大量连续 case = 跳转表
- for/while 循环的固定汇编结构
- 数组访问：`基址 + 索引 * sizeof` 的汇编形态
- 函数调用完整流程：push 参数（从右到左）→ call → 建栈帧 → 执行 → 返回值 eax → add esp 清理
- 调用约定差异：cdecl vs stdcall vs fastcall

**第 4 章 C++↔汇编：**
- this 指针 = ecx，成员访问 = `[ecx + 偏移]`
- 虚函数表：对象头部 4 字节是 vtable 指针
- 虚函数调用：`mov edx, [eax]` → `call [edx + 偏移]`
- 继承内存布局，多重继承的多 vtable 指针
- new/delete 的底层（operator new → 构造函数）
- STL 容器的汇编特征（string、vector）

### 第 5 章 CE 精讲

- 精确数值搜索 vs 未知初始值搜索
- 指针扫描原理与操作
- 找基址 + 偏移的完整流程
- 代码注入（Code Injection）操作步骤
- AOB 注入（Array of Bytes）操作步骤

### 第 6 章 CrackMe 递进精讲

三个 CrackMe 难度递进：
1. **简单字符串比较**：直接从内存读到注册码
2. **简单数学运算**：跟踪算法（异或、位运算等），手工算出注册码
3. **稍复杂算法**：写出注册机程序（Keygen）

### 第 9-10 章 终极项目精讲

完整的 DLL 注入 + Hook 开发流程：
- 用 CE 确认地址和偏移
- 用 C/C++ 编写 DLL
- 编写注入器（CreateRemoteThread）
- Hook 技术（Inline Hook / IAT Hook）
- 调试注入后的 DLL
